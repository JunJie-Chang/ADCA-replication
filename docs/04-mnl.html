4.1 Specification

Utility of alternative \(j\) in choice situation \(i\): \[ V_{ij} = \beta_j + \alpha_j P_{ij}, \qquad j \in \{\text{GEN, GT, NAT, CHO, CAB}\}, \quad V_{i,\text{NONE}} \equiv 0 . \] There are 5 alternative-specific constants \(\beta_j\) and 5 own-price coefficients \(\alpha_j\) (10 taste parameters). The joint model stacks both treatments, holds taste parameters equal, and scales the nonhypothetical utilities by \(\mu\): \[ V^{\text{NH}}_{ij} = \mu\,(\beta_j + \alpha_j P_{ij}), \qquad V^{\text{H}}_{ij} = \beta_j + \alpha_j P_{ij}, \] so \(\mu\) is the relative error scale (precision) of the real-payment group. The Swait–Louviere LR test compares the joint model against the two separate models.

4.2 Estimation code (R/03_table3_mnl.R)

# Alternative-specific price variables: p_j = price × 1{alt = j}
df0 <- df_mlogit |>
  mutate(price_alt = tidyr::replace_na(price_alt, 0),
         p_GEN = if_else(alt=="GEN", price_alt, 0),
         p_GT  = if_else(alt=="GT",  price_alt, 0),
         p_NAT = if_else(alt=="NAT", price_alt, 0),
         p_CHO = if_else(alt=="CHO", price_alt, 0),
         p_CAB = if_else(alt=="CAB", price_alt, 0))

make_mdata <- function(d) mlogit.data(d, choice="chosen", shape="long",
  alt.var="alt", chid.var="chid", id.var="subject", alt.levels=alts)

f_mnl <- chosen ~ p_GEN + p_GT + p_NAT + p_CHO + p_CAB | 1   # 5 prices + 5 ASCs
mnl_nh  <- mlogit(f_mnl, make_mdata(filter(df0, treatment=="nonhyp")), reflevel="NONE")
mnl_hyp <- mlogit(f_mnl, make_mdata(filter(df0, treatment=="hyp")),    reflevel="NONE")

# Joint model with scale mu: custom log-likelihood maximised by optim(BFGS).
# theta = (beta[1:5], alpha[6:10], log_mu[11]); V_NH scaled by exp(log_mu).
ll_joint_fn <- function(theta, df) {
  beta  <- c(theta[1:5], 0); alpha <- c(theta[6:10], 0); mu <- exp(theta[11])
  k <- match(as.character(df$alt), alts)
  V <- beta[k] + alpha[k]*df$price_alt
  V[df$treatment=="nonhyp"] <- mu * V[df$treatment=="nonhyp"]
  df |> mutate(V=V) |> group_by(chid) |>
    mutate(p = exp(V-max(V))/sum(exp(V-max(V)))) |> ungroup() |>
    summarise(ll = sum(log(pmax(p[chosen], 1e-300)))) |> pull(ll)
}
opt <- optim(theta0, ll_joint_fn, df=df0, method="BFGS",
             control=list(fnscale=-1, maxit=1e4, reltol=1e-12), hessian=TRUE)

# Swait–Louviere LR scale test: chi2 = -2(LL_joint - LL_nh - LL_hyp)
lr_stat <- -2 * (opt$value - logLik(mnl_nh) - logLik(mnl_hyp))

Discrepancy — scale \(\mu\) and test df

My estimated scale is \(\hat\mu = 0.804\) (SE \(0.056\)), versus the paper’s \(0.90\) (SE \(0.06\)). Both are statistically distinguishable from 1 and point the same way (the two treatments have different error scales), but the point estimates differ. My scale test gives \(\chi^2 = 105.9\) on 9 df versus the paper’s \(106.8\) on \(4\) df. The \(\chi^2\) magnitudes are close; the df differ because of how the joint restriction is counted (I restrict 10 taste parameters to be equal and free 1 scale: \(df = (10+10) - 11 = 9\); the paper’s “4” suggests different bookkeeping). The substantive conclusion — reject pooling — is unchanged.

Note also that my model log-likelihoods sit below the paper’s (nonhyp \(-1400.3\) vs \(-1261.4\); hyp \(-677.9\) vs \(-627.4\)). This level offset recurs in the IIA test and is the main open question of the replication — most plausibly a difference in how price enters utility. See Discrepancies.

## Specification

Utility of alternative $j$ in choice situation $i$:
$$
V_{ij} = \beta_j + \alpha_j P_{ij}, \qquad j \in \{\text{GEN, GT, NAT, CHO, CAB}\},
\quad V_{i,\text{NONE}} \equiv 0 .
$$
There are **5 alternative-specific constants** $\beta_j$ and **5 own-price
coefficients** $\alpha_j$ (10 taste parameters). The **joint model** stacks both
treatments, holds taste parameters equal, and scales the nonhypothetical
utilities by $\mu$:
$$
V^{\text{NH}}_{ij} = \mu\,(\beta_j + \alpha_j P_{ij}), \qquad
V^{\text{H}}_{ij}  =      \beta_j + \alpha_j P_{ij},
$$
so $\mu$ is the relative error scale (precision) of the real-payment group. The
**Swait–Louviere LR test** compares the joint model against the two separate
models.
